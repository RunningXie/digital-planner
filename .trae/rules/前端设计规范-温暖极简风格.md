# Dear Diary 前端设计规范（温暖极简风格）

所有页面、组件、CSS 修改必须遵循以下规范，保持整套产品视觉一致。

## 一、整体定位

- **风格**：温暖极简（Warm Minimalism）
- **关键词**：克制 / 安静 / 阅读友好 / 不喧宾夺主
- **核心原则**：内容是主角，UI 退到背景；一种主色，一个字体家族，三档圆角

## 二、色彩系统（CSS 变量，定义在 `static/css/style.css`）

### 背景
- `--bg: #f7f4ef` — 页面背景（暖米白）
- `--surface: #ffffff` — 卡片表面
- `--surface-2: #faf8f4` — 输入框 / hover

### 文字
- `--text: #2e2823` — 主文字（深棕）
- `--text-2: #6b6258` — 次要文字
- `--text-3: #9a9088` — 三级（占位、辅助）

### 边框
- `--border: #e8e2d8`
- `--border-strong: #d8d0c4`

### 强调色（克制使用，全站只此一种）
- `--accent: #8b6f5c` — 暖棕：主按钮、链接
- `--accent-hover: #6e5746`
- `--accent-soft: #f3ede4`

### 状态色
- 成功 `--success: #5d8a6a` / `--success-soft: #ebf2ed`
- 错误 `--error: #b95a4a` / `--error-soft: #f7ebe8`
- 警告 `--warning: #b8884a` / `--warning-soft: #f6efe2`

### 规则
- **禁止**新增色相（蓝、绿、紫、彩虹渐变等）
- **禁止**使用纯黑 `#000` 或纯白 `#fff` 文字
- 强调色一次页面最多出现 1–2 处

## 三、圆角与阴影

- 三档圆角：
  - `--radius-sm: 6px` — 标签、徽章
  - `--radius: 8px` — 按钮、输入框、卡片
  - `--radius-lg: 12px` — 模态框、大卡片
- 三档阴影：
  - `--shadow-sm` — 微妙浮起
  - `--shadow` — 默认卡片
  - `--shadow-md` — 弹层、Toast
- **禁止**用大投影、霓虹光晕、彩色阴影

## 四、字体

- 主字体（`--font-sans`）：系统字体栈 + PingFang SC / 微软雅黑
- 衬线（`--font-serif`）：仅在日记正文、引言等需要"书卷气"时使用
- 基础字号：`0.9375rem`（15px）
- 标题：h1 1.625 / h2 1.25 / h3 1.0625 / h4 0.9375
- 行高：`1.6`（正文）、`1.3`（标题）
- 字重：标题 600，正文 400，最多用到 600，**不要用 700+ 加粗**

## 五、图标

- **统一使用 SVG 行内图标**（stroke-width 1.6–1.8，linecap/linejoin round）
- **完全禁止 emoji 装饰**（✍️📓🌧️☀️ 等都不允许出现在 UI 上）
- 导航、按钮、统计卡片标题前可配 18–24px 的 SVG 图标
- 图标颜色用 `currentColor` 跟随文字色

## 六、按钮

使用 `.btn` 体系，文字必须一行显示：

- `.btn-primary` — 主操作（accent 背景 + 白字）
- `.btn-secondary` — 次要操作（surface 背景 + 边框）
- `.btn-ghost` — 文字按钮（无背景）
- `.btn-danger` — 危险操作（error 色）
- 高度：38–40px，圆角 `--radius`，padding 0.5–0.75rem
- **按钮文字禁止换行**（用 `white-space: nowrap` 保证单行）

## 七、布局

- 容器最大宽度 1200px（`--container` / `.container`）
- 页面内容容器：`.container`（默认）、`.container-wide`（后台管理）
- 页面主 padding：top 1.5–2rem，左右 1.5rem
- 区块间距：1.5rem
- 卡片间距：0.75–1rem
- 响应式断点：
  - ≥ 768px：桌面布局
  - < 768px：单列堆叠，导航收起

## 八、动效

- 过渡时间：0.15–0.2s
- 缓动：`ease` / `ease-out`
- **禁止**复杂动效：
  - 不做天气动画、粒子、背景漂浮
  - 不做夸张的 hover 缩放、旋转
  - 不做翻页动画
- 允许的动效：按钮 hover 颜色、Toast 淡入淡出、列表项出现的极短渐变

## 九、组件约定

- **导航栏 `.navbar`**：sticky 在顶部，56px 高，surface 背景，下边框
- **卡片 `.card` / `.stat-card`**：surface 背景，1px border，radius
- **统计网格 `.stat-grid`**：grid 布局，minmax 200px
- **表格 `.table`**：行 hover surface-2，斑马纹禁用
- **Token 进度条 `.token-bar`**：高度 6px，accent 填充，文字在外
- **Toast `.toast-message`**：右下角浮层，自动 2–3s 消失
- **表单**：label 在上、输入框在下，间距 0.375rem
- **空状态**：居中插画 + 引导文字 + 主按钮

## 十、页面结构模板

所有新页面必须：

1. 继承 `{% extends "base.html" %}`（有导航的页面）
2. 标题前用 SVG 图标，不用 emoji
3. 标题字号 1.375rem，weight 600
4. 使用 `extra_css` 块写页面专属 CSS（不污染全局）
5. 使用 `extra_js` 块写页面脚本
6. 统一 token 获取：`localStorage.getItem('diary_token')`
7. 401 时统一跳 `/login`

## 十一、内容文案

- 中文为主，英文为辅（产品名 Dear Diary、按钮 "Get Started" 等例外）
- 语气：温和、不催促、像写作伙伴
- 错误提示：用 `error-soft` 底色 + `error` 文字，**不要用 alert 弹窗**

## 十二、禁止事项清单

- ❌ emoji 装饰（任何位置）
- ❌ 天气、季节、节日等主题动效
- ❌ 渐变背景、玻璃拟态、霓虹色
- ❌ 圆角超过 16px
- ❌ 字体大小超过 2rem（hero 区可到 2.5rem）
- ❌ 按钮文字换行
- ❌ 标题输入框（已统一移除）
- ❌ 翻页器 / 复杂分页
- ❌ 在模板里直接写 `<style>` 改全局变量

## 十三、修改前自检

每次改前端前问自己：

1. 颜色是不是只用了 token 里的？
2. 圆角用的是不是 6/8/12？
3. 有没有不小心引入 emoji？
4. 动效是不是在 0.2s 以内的简单过渡？
5. 手机端（≤375px）会不会错位？
6. 文字长度翻倍时按钮会不会撑开？
